# BookLink (Snap Trade) 기획 및 서비스 개요 파헤치기

현재 구현된 `BACKEND_현재구조_해부.md`와 `FRONTEND_현재구조_해부.md`의 구조와 API 목록을 종합해 볼 때, **BookLink**는 단순한 중고책 거래 앱을 넘어선 **"통합 도서 탐색 및 간편 거래 플랫폼"**을 지향하고 있습니다.

어떤 서비스를 만들고자 했는지 구체적인 기획 의도와 핵심 기능을 역추적하여 정리했습니다.

---

## 1. 서비스의 정체성 (Identity)

### 📸 **"찍어서 바로 파는" 중고책 마켓 (Snap Trade)**

가장 눈에 띄는 차별점은 **OCR(광학 문자 인식) 기능**입니다. 사용자가 책등이나 표지 사진을 찍어 올리면, Google Vision API와 클라이언트 Tesseract.js를 활용해 책 정보를 추출(ISBN, 제목, 저자 등)하여 판매 등록 폼을 자동으로 채워줍니다. 귀찮은 텍스트 입력 과정을 생략하여 중고책 판매의 진입장벽을 크게 낮추려는 의도가 보입니다.

### 🔍 **"살까, 빌릴까, 중고로 살까?" - 올인원 도서 검색 허브**

단순히 유저 간의 중고 거래만 연결하는 것이 아니라, 특정 책을 찾을 때 **모든 대안을 한 번에 제시**하는 강력한 검색 페이지(`/search`)를 기획했습니다.

- **buy (책 사기):** 카카오 API(새 책), 알라딘 마켓(온라인 중고), 북링크 유저 마켓(개인 간 중고), 알라딘 오프라인 매장(크롤링 기반 반경 필터링) 정보를 한눈에 비교.
- **borrow (책 빌리기):** 사용자 위치(Geolocation)를 기반으로 근처 도서관(하버사인 거리 계산)을 탐색하고 장서 소장 여부까지 확인.

## 2. 유저 저니(User Journey)와 핵심 기능

### A. 책을 판매하려는 유저 (Seller)

1. **사진 등록 (Snap!):** `/sell` 페이지에서 책을 카메라로 촬영하거나 이미지를 업로드합니다.
2. **자동 정보 추출:** 프론트엔드가 백엔드의 `/api/ocr/extract`를 호출하여 이미지 내의 글자를 읽습니다.
3. **정보 보정:** 추출된 텍스트를 바탕으로 알라딘/카카오 API를 통해 정확한 도서 메타데이터(카테고리, 표지 이미지 등)를 매핑합니다.
4. **판매 등록:** 책 상태(Condition), 위치(Location), 가격(Price)을 입력하여 `Sale` 엔티티로 등록합니다. (현 상황: UI까지는 구현되었으나 백엔드 저장 API 연동은 개발 중)

### B. 책을 구하려는 유저 (Buyer / Borrower)

1. **통합 검색:** `/search` 페이지에서 찾고 싶은 책을 검색합니다.
2. **다양한 선택지 비교:**
   - **무료로 빌리고 싶다면:** 내 주변 로컬 도서관의 대출 가능 여부 확인 (`POST /api/library/search`).
   - **가장 싸게 사고 싶다면:** 유저들이 올린 매물 목록(`Marketplace`)과 알라딘 오프라인 중고 매장(`POST /api/aladin/used-store`)의 재고 및 최저가를 비교합니다.
3. **거래 및 찜하기:** 원하는 유저 매물을 발견하면 상세 페이지(`/book/:id`)에서 확인 후 찜(Wishlist)하거나 거래를 진행합니다.

### C. 마이페이지 관리 (My Page)

1. **인증:** JWT 기반 로그인(`POST /api/auth/login`)을 통해 안전하게 앱을 이용합니다.
2. **대시보드:** 내가 등록한 판매 내역(`SalesHistory`), 구매 내역(`PurchaseHistory`), 찜한 목록(`WishList`)을 한곳에서 모아보고 거래 상태를 관리합니다.

## 3. 기획자가 의도한 비즈니스 로직 (아키텍처 관점)

- **위치 기반 서비스(LBS):** 프론트엔드에서 Geolocation으로 좌표를 가져오고, 백엔드에서는 Haversine 공식으로 내 주변의 알라딘 중고매장이나 도서관을 필터링합니다. 이는 "내 동네 프렌들리"한 오프라인 연계 경험(O2O)을 노린 것입니다.
- **크롤링 기반 데이터 통합:** 알라딘 중고매장 데이터는 공식 API의 한계를 극복하고자 HTML 크롤링(jsoup)을 활용해 매장별 정확한 가격과 재고를 파싱하여 제공하려 했습니다.
- **확장성을 고려한 외부 연동:** OCR, 카카오, 알라딘, 공공 도서관 등 다수의 외부 리소스를 끌어와 데이터를 정규화(DTO)하는 것에 백엔드의 많은 역할이 집중되어 있습니다.

---

## 4. 현재 개발 현황 및 MVP (Minimum Viable Product) 도달을 위한 과제

현재 프론트엔드와 백엔드의 뼈대는 훌륭하게 설계되어 있으나, 이 서비스가 "진짜 돌아가는(Woring) 앱"이 되기 위해 남은 병목 지점들은 다음과 같습니다.

1. **검색 연동의 완성:** 프론트엔드의 `Search.tsx`에서 병렬로 호출하는 외부 검색(알라딘, 카카오) 로직이 현재 더미 에러를 던지고 있습니다. 이 API들을 백엔드에 완성하거나 정상적으로 Mocking 해야 합니다.
2. **코어 트랜잭션 연결:** 핵심 기능인 `/sell`(판매 등록) 시 백엔드 `POST /api/books` 및 `POST /api/sales`를 호출하여 실제 MySQL DB(`booklink`)에 적재하는 마무리 작업이 필요합니다.
3. **소셜 로그인 (OAuth):** 구글/카카오 로그인 버튼은 존재하지만 백엔드는 `501 Not Implemented` 상태입니다.
