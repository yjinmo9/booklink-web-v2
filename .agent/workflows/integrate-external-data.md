# External Data Integration Workflow

새로운 외부 데이터 소스를 우리 프로젝트에 안전하게 통합하는 표준 프로세스입니다.

## 1. 외부 인터페이스 명세 분석

1. 사용하고자 하는 API의 호출 URL, 인증 방식(API Key), 응답 스펙(JSON/XML)을 분석합니다.

## 2. API 전용 클라이언트 구축 (TDD)

1. `.agent/skills/generate_external_api_client` 스킬을 실행하여 통신 계층 뼈대를 만듭니다.
2. 외부 서비스가 점검 중이거나 장애가 났을 때를 대비한 **실패 테스트 케이스(Timeout, 5xx Error)**를 먼저 작성합니다.
3. 실제 응답을 내부 도메인 객체로 변환하는 매핑 로직을 구현합니다.

## 3. 내부 서비스 연동 및 캐싱

1. 외부 데이터를 가져오는 로직이 너무 느린 경우, 캐싱 전략(Redis 또는 로컬 캐시)을 검토하여 적용합니다.
2. 통합 테스트를 통해 데이터 정규화가 의도대로 되었는지 최종 확인합니다.
