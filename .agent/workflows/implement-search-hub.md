# Implement Search Hub Workflow

`/search` 페이지의 핵심인 다중 소스 통합 검색(Aggregator) 기능을 개발하는 절차입니다.

## 1. 검색 소스 식별

1. 새 책(카카오), 온라인 중고(알라딘), 유저 매물(Local DB), 오프라인 매장(크롤링) 등 호출 대상을 정리합니다.

## 2. 병렬 호출 아키텍처 구현

1. 각 검색 소스를 순차적으로 호출할 경우 응답 시간이 매우 길어지므로, 자바의 `CompletableFuture` 등을 이용해 **비동기 병렬 호출** 구조를 설계합니다.
2. 모든 소스로부터 결과를 취합(Merge)하고, 중복을 제거하거나 정렬하는 로직을 구현합니다.

## 3. 위치 기반 필터링 적용

1. 사용자의 현재 좌표 정보를 받아서 하버사인 공식을 적용, "내 주변" 매물이나 오프라인 매장만 필터링하도록 고도화합니다.

## 4. UI 연동 및 결과 시각화

1. 프론트엔드(`Search.tsx`)와 연동하여 병렬 호출된 결과가 화면에 끊김 없이 렌더링되도록 확인합니다.
