# LBS & Crawling Rules

## 1. 위치 기반 서비스 (LBS)

- **표준 거리 계산**: 모든 거리 계산은 Haversine 공식을 사용하거나, PostGIS와 같은 공간 인덱스 기능을 활용할 경우 해당 라이브러리의 표준 함수를 사용합니다.
- **좌표 검증**: 클라이언트에서 전달된 위경도 좌표값이 유효한 범위(-90~90, -180~180)인지 반드시 검증합니다.

## 2. 웹 크롤링 (JSoup 등)

- **격리된 파서**: HTML 태그를 분석하는 로직(`Element.select` 등)은 별도의 `Parser` 클래스로 분리합니다. (사이트 개편 시 유지보수 용이성)
- **사용자 에이전트(User-Agent)**: 모든 크롤링 요청에는 적절한 User-Agent 헤더를 설정하여 차단을 방지합니다.
- **에러 핸들링**: 특정 셀렉터가 찾지 못할 경우(`NullPointerException` 방지)에 대한 기본값 처리를 확실히 합니다.
