# Git Commit Convention

본 프로젝트는 협업과 히스토리 추적의 용이성을 위해 `Conventional Commits` 규격을 기반으로 한 커밋 메시지 작성을 원칙으로 합니다. 안티그래비티는 모든 자동/수동 커밋 시 이 규칙을 준수합니다.

## 1. 메시지 구조

```text
<type>: <subject>

<body> (선택사항)

<footer> (선택사항)
```

## 2. 타입 (Types)

- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **docs**: 문서 수정 (README.md, .agent/\*.md 등)
- **style**: 코드 포맷팅, 세미콜론 누락 등 (비즈니스 로직 변경 없음)
- **refactor**: 코드 리팩토링 (기능 변화 없이 구조 개선)
- **test**: 테스트 코드 추가 및 수정
- **chore**: 빌드 업무 수정, 패키지 매니저 설정, 프로젝트 설정 변경 등

## 3. 모범 사례

- 제목은 50자 이내로, 영문 기준 명령문 형식을 사용하거나 명확한 한글 요약을 사용합니다.
- 안티그래비티가 작업 완료 후 커밋할 때마다 이 문서를 참조하여 자동으로 메시지를 생성합니다.
